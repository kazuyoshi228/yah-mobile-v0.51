# 購入ドロワー：ログイン画面から期間・データ量を変更できるようにする UX 提案

**対象:** `client/src/components/app/PurchaseDrawer.tsx` および `purchase-drawer/steps/*`
**作成日:** 2026-07-03

---

## 1. 課題

購入フローは 6 ステップ構成:

```
DURATION → DATA → PRICE → LOGIN → PAYMENT → ESIM
 (step0)   (step1) (step2) (step3) (step4)  (step6)
```

**LOGIN（step3）の「未ログイン」画面には「戻る」導線が無い。**
そのため、ここまで来たユーザーは「やっぱり期間やデータ量を変えたい」と思っても、

- 一度サインインするしかない、または
- ドロワーを閉じて最初からやり直すしかない

という状態で、離脱の原因になりうる。

### 現状の各ステップの「戻る」導線

| Step | 画面 | 戻る導線 |
|---|---|---|
| 0 | DURATION（期間選択） | ドロワーを閉じる（Back = Close） |
| 1 | DATA（GB選択） | Back → step0 |
| 2 | PRICE（価格確認） | Back → step1 |
| **3** | **LOGIN（未ログイン）** | **なし** ← 🔴 課題 |
| 3 | LOGIN（ログイン済み） | Back → step2 |
| 4 | PAYMENT（同意・決済） | Back → step2 |

上部の**ステップインジケーター**（`✓ DURATION — ✓ DATA — ✓ PRICE — 4 LOGIN …`）は
現状**表示専用でクリックできない**。

---

## 2. 提案（3案）

### 案A（推奨・汎用）— ステップインジケーターをクリック可能にする
上部インジケーターの**完了済みステップ（DURATION / DATA / PRICE）をクリックで戻れる**ようにする。

- **メリット**
  - ログイン画面に限らず、**どのステップからでも**任意の完了ステップへ即移動できる
  - 実装が**1箇所（インジケーター）**で全体に効く／UI を増やさない
  - 「今どこにいて、どこに戻れるか」が一目で分かる（既に見えているUIを活用）
- **配置**: 既存のステップインジケーター（画面上部）
- **挙動**: 完了ステップ（`i < step`）のみクリック可 → `setStep(i)`。未来のステップは無効（グレーのまま）
- **見た目**: 完了ステップにホバー下線＋`cursor: pointer`

### 案B — ログイン画面に「戻る」ボタンを追加
未ログイン branch の下部に「← プランを変更」ボタンを追加し、PRICE（step2）へ戻す。

- **メリット**: 実装が局所的で明快。ログイン画面での離脱を直接的に減らせる
- **デメリット**: この画面だけの対処（案A が無いと他ステップと非対称）
- **配置**: サインインカードの直下、左寄せ（アウトライン系ボタン）
- **挙動**: `setStep(2)`（そこからさらに Back で Data / Duration へ辿れる）

### 案C — 選択中プランのサマリー＋「変更」リンクを常設
各ステップ上部に「7日 / 3GB / ¥1,350 [変更]」のような**選択内容サマリー**を常設。

- **メリット**: 選択内容が常に見えて安心感がある／変更導線が明確
- **デメリット**: UI の追加とデザインが必要（情報量が増える）
- **挙動**: [変更] → DURATION（step0）へ

---

## 3. 推奨案

> **案A（インジケーターのクリック化）＋ 案B（ログイン画面の戻るボタン）の併用**

- **案A** … 最小実装で「完了ステップへ戻れる」を**全体**に付与（最大効果）
- **案B** … ログイン画面に**明示的な戻る導線**（未ログインユーザーの離脱防止）
- **案C** … 将来の強化（サマリー常設）として保留

理由: 案A だけでも解決するが、ログイン画面は「別画面へ飛ぶ（Googleログイン）」直前で離脱が起きやすいため、**その画面内に明示ボタン（案B）**もあると親切。両者は competing せず相互補完。

## 4. 具体的なボタン配置イメージ

### ログイン画面（step3・未ログイン）

```
┌───────────────────────────────────────┐
│              [🔓 サインインカード]          │
│      Sign in to continue（Googleログイン）  │
│         利用規約・プライバシー同意文           │
└───────────────────────────────────────┘

   [← プランを変更]        ← 案B：新規追加（左寄せ・アウトライン）

──────────────────────────────
 01  Purchase history saved to your account
 02  QR code sent to your registered email
 03  Full support access with history
```

- ラベル案: **「← Change plan / ← プランを変更」**
- 挙動: `setStep(2)`

### ステップインジケーター（全ステップ共通・案A）

```
[✓ DURATION] — [✓ DATA] — [✓ PRICE] — [4 LOGIN] — 5 PAYMENT — 6 ESIM
   ↑クリック可     ↑クリック可   ↑クリック可    ↑現在地      ↑無効       ↑無効
```

- 完了済み（✓）ラベル: ホバーで下線、`cursor:pointer`、`onClick → setStep(i)`
- 現在地・未来: 変更なし（クリック不可）

## 5. 実装の要点（参考）

| 変更対象 | ファイル | 内容 |
|---|---|---|
| 案A: インジケーターのクリック化 | `PurchaseDrawer.tsx`（`drawerStepLabels.map` の部分） | 完了ステップの `<span>` を `<button>` 化し `onClick={() => setStep(i)}`（`i < step` のみ有効） |
| 案B: ログインの戻るボタン | `purchase-drawer/steps/Step3Login.tsx`（未ログイン branch） | サインインカード下に Back ボタン追加 → `setStep(2)`。`setStep` は Context (`usePurchaseDrawerCtx`) から取得済み |

### 注意点
- 戻ってプランを変更しても、同意チェック等の下流状態は保持でOK（PAYMENT で再表示・再確認される）
- 戻り先を PRICE（step2）にするのが安全（`currentOpt` が有効な状態）。そこから Back で DATA / DURATION へ辿れる
- i18n キー（例: `drawer.changePlan`）を1つ追加する想定
